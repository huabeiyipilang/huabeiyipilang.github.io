公司刚从svn转到git，今天简单的看了两章progit.pdf，做一下笔记。

 

本地项目目录项的文件分两种：

  • untracked files: 未被追踪的文件，也就是未被git管理的文件
  • tracked files：被追踪的文件，也就是被git所管理的文件

git所管理的文件（tracked files）有三种状态：

  • unmodifid/commited: 未修改，或者说已提交
  • modified:已修改
  • staged: 已暂存

实例流程：

  1. vi README，在项目中新建一个文件README，（untracked files）
  2. git add README， 将README纳入git管理范围。（staged）
  3. git commit -m "add new file README"，提交文件。（commited）
  4. vi README，修改文件。 （modified）
  5. git add README，将修改文件暂存 。（staged）
  6. git commit -m "modify README"，提交文件 （commited）
  7. git rm README，从git中删除文件（untracked ）

撤销操作：

  • 撤销修改：git checkout
  • 撤销暂存：git reset
  • 没有撤销提交的操作，只有补充提交：git commit -amend，将本次提交合并到上次提交中
